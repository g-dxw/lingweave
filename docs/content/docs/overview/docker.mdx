# Docker 部署

如果你希望在自己的机器或服务器上运行项目，可以直接使用 Docker Compose。

## 使用发布镜像

```bash
git clone git@github.com:g-dxw/lingweave.git
cd lingweave
docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

## 本地构建镜像

如果需要基于当前源码构建镜像：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

镜像默认不启用 Niffler。需要在自定义部署中启用时，在构建前设置浏览器可访问的地址：

```bash
VITE_NIFFLER_ORIGIN=https://niffler.example.com docker compose -f docker-compose.local.yml up -d --build
```

该地址会写入前端静态产物，修改后需要重新构建镜像。
如果填写跨域地址，对应 Niffler 环境需要允许浏览器跨域凭证请求。

## 文档站镜像

文档站位于 `docs/` 目录。根目录主应用镜像会先将文档静态编译，再映射到主应用的 `/docs/`，因此默认不需要额外启动文档容器。

如果只需要单独部署文档，也可以使用文档镜像：

使用发布镜像：

```bash
cd docs
docker compose up -d
```

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据说明

当前主应用镜像使用 nginx 托管主应用和已编译文档。画布、我的素材、生成记录和 AI API Key 默认保存在浏览器本地；提示词库和帮助文档随静态产物发布，不需要额外挂载数据目录。
