# 待测试

- AI 思维节点：支持从工具栏、空白画布菜单或已有素材连线创建，按自由发散、连贯序列、总分展开生成 2～8 个不同文本子节点，并兼容从 Responses 流事件返回的函数调用结果；重复运行保留旧结果，需验证文本与图片输入、三种布局、停止请求、浅色/深色主题以及子节点继续连接生成配置。
- 生成配置提示词：引用素材按组装模板中的首次出现顺序编号，最终提示词分为任务说明、参考素材和引用文本，文本内容保持原文且图片请求不再重复追加编号说明；需验证单个、多图片、多文本和重复引用场景下的预览、复制、生成与重试内容一致。
- 项目品牌：页面、文档、插件展示名和 Agent 提示统一更新为 `LingWeave（灵织）`，主站、文档和 Codex 插件使用交织双线品牌图标，版本基线重置为 `v0.1.0`，GitHub 与 Docker 地址切换到 `g-dxw/lingweave`；本地存储和 MCP 技术标识保持不变，需验证页面标题、导航、浏览器图标、版本检查、文档、插件说明、Niffler Key 创建和旧数据读取。
- Niffler 环境配置：本地开发默认将 `/api`、`/v1` 代理到 `https://niffler.org`，可用 `NIFFLER_PROXY_TARGET` 覆盖；生产构建未配置 `VITE_NIFFLER_ORIGIN` 时隐藏登录入口并清空默认 Niffler Base URL，配置后改用对应地址，需分别验证开发、GitHub Pages 和自定义部署。
- 画布连续生成：直接从已有图片、视频、音频或文本节点继续生成时，不再读取其下游生成配置的提示词；从生成配置执行时仍使用该配置的上游提示词和参考素材，需验证单个与多个下游配置场景。
- 画布节点提示词：输入框获得焦点后显示主题一致的边框与聚焦环，需验证浅色、深色主题以及资源引用高亮状态下均清晰可见。
- 首屏性能：路由页面与全站 Agent 面板改为按需加载，Agent 的 Streamdown 样式随面板加载；需验证各路由直达与切换、首次打开和关闭 Agent、携带 Agent 参数进入画布均正常。
- 画布保存：节点、连线、对话、背景、图片信息和视口统一防抖保存，并按画布项目独立写入浏览器存储；需验证连续拖动、刷新恢复、切换画布、删除画布和导入同步不会丢失数据。
- 依赖清理：移除未使用的 Ant Design Pro、Ant 图标和直接 dayjs 依赖，将 shadcn 调整为开发依赖并更新锁文件；需验证主题、GitHub 图标、npm 与 Bun 安装流程。
- 自动化测试：前端接入 Vitest、Happy DOM 和 GitHub Actions，新增输入法键盘、API URL 与模型渠道、Niffler 有效配置、默认案例安装、提示词加载筛选与分页测试；Test workflow 会在 push 和 pull request 后执行测试、类型检查、文档构建和 Web 构建，需验证本地命令和远端 workflow。
- 配置与用户偏好：启用 Niffler 的环境提供登录入口和对应默认 Base URL；未启用时默认渠道地址为空，用户仍可填写自己的 Base URL 和 API Key，需验证环境默认值不会覆盖自定义渠道。
- 默认画布案例：首次打开站点会自动安装“电动越野摩托车全场景创作”案例，包含产品主视觉、森林穿越、城市穿行三组提示词、配置、参考图和生成结果；需验证首次安装、图片显示、节点连线、刷新不重复安装，以及删除后从案例入口可重新安装。
- 项目信息：移除商务合作、打赏和广告赞助内容，仓库、部署、插件、版本检查及文档链接更新为 `g-dxw/lingweave`；需验证页面导航、帮助文档和外部链接。
- 首页与使用指南：独立首页按连续视觉创作重新设计，以电动越野摩托车案例说明产品价值；独立 `/guide` 快速上手不再重复案例和功能介绍，改为模型准备、文本节点一键生图、结果引用与常见问题指南，需验证桌面端、移动端及画布、配置、案例入口。
- 提示词库资源：本地静态库不再提交到 Git；开发和构建时优先下载当前版本 GitHub Release 资源包，不可用时从上游同步，版本标签发布时自动生成并上传资源包；需验证干净环境首次启动、Release 下载、上游兜底、人工更新、筛选分页和封面失败占位。
- 提示词库标签：同步时将模型、作者和仓库来源从内容标签中剥离，合并常见中英文同义词并归入固定创作分类；页面和画布选择弹窗默认展示 16 个高频标签，更多标签支持搜索，卡片最多展示 3 个标签，需验证分类数量、标签计数、组合筛选和窄屏布局。
- 登录与模型配置：未登录不再被登录页阻断，可填写 Base URL 和 API Key、拉取并选择模型；登录 Niffler 后可选择账号 API Key 和各类默认模型，模型选择刷新后保留，账号 Key 只保留在页面运行内存，本地自定义渠道不会被覆盖；需验证登录、退出、Key 切换、模型拉取、模型分类、Agent 配置读取和两种状态下的生成请求。
- 画布节点创建：鼠标左键双击画布空白区域会在点击位置打开节点选择菜单，可创建文本、图片、视频、音频、生成配置和组节点；双击已有节点或连线不应触发菜单。
- 本地 Agent 连接说明：新增提醒说明只有安装 Codex 插件或手动添加 MCP 后才会增加 Codex token 消耗，直接运行 `npx -y @basketikun/canvas-agent` 不会安装 MCP；需验证文案位置清晰。
- 本地 Agent 连接说明：画布面板和配置页 Codex Tab 改为区分“Codex 插件启动画布”和“直接运行 Agent 后网页连接”两种方式，需验证文案和布局清晰。
- 全站 Agent：Agent 面板从画布页抽离为全站常驻面板，挂在全局布局右侧，开/关只挤压左侧内容不遮挡；顶栏和画布工具条各有一个开关按钮，需验证在首页、生图、视频、素材、提示词库、画布等页面都能打开面板并保持 SSE 连接不断开。
- 全站 Agent：新增 `site_navigate` 工具，Agent 可直接跳转 `/`、`/canvas`、`/canvas/:id`、`/image`、`/video`、`/prompts`、`/assets`、`/config`；导航直接执行不走工具确认，需验证跳转正确且面板不消失。
- 全站 Agent：画布工具（canvas_apply_ops 等）仅在当前处于画布页时可用，不在画布页调用会返回“当前不在画布页，请先用 site_navigate 打开画布”，需验证 Agent 会先导航到画布再执行画布操作。
- 全站 Agent：`canvas_generate_*` 已包含自动生成，Agent 不再额外追加 `canvas_run_generation`；同一节点生成中收到重复触发会返回成功但不创建第二个结果节点，需验证图片、文本、视频、音频首次生成、运行中重复触发、停止后重跑和完成后再次生成。
- 全站 Agent：Codex 会话改为单一站点级连续线程（不再按 canvasId 分），跨页面/跨画布保持同一条对话上下文，历史记录即这条线程；本地 Agent workspace 从按画布分改为固定 `site` workspace（`~/.infinite-canvas/codex-workspaces/site`），已装 `@basketikun/canvas-agent` 的用户需更新到新版本才生效。
- 全站 Agent：移除仅前端调用 OpenAI responses 接口的在线画布助手死代码（含 `requestToolResponse`），Agent 统一走 MCP + 本地 Codex 链路。
- 画布节点名称：每个节点保留 `title` 名称字段，默认名称显示在节点上方，双击名称可直接编辑，节点信息弹窗和 JSON 可查看该字段。
- 画布节点工具条：节点顶部工具条改为点击选中节点后显示，单纯鼠标移入 / 移出节点不应再频繁弹出。
- 画布组节点：组内子节点优先响应点击和拖拽；当鼠标位于组内节点上时，应移动 / 编辑该节点，而不是移动整个组。
- 画布组节点：工具栏新增“组”节点，普通节点拖入组区域会高亮目标组，松手后自动吸附进组；拖出组区域会解除归属，拖动组节点会一起移动组内子节点。
- GitHub Pages：`main` 分支合并和版本 tag 均会触发前端静态站点自动发布 workflow，应用路由、文档、搜索和模型图标均适配仓库子路径；需验证 PR 合并后自动部署，以及首页、路由刷新和文档跳转。
- 画布图片切图：行数 / 列数会生成可直接拖拽的等分切图线，并支持新增横向 / 纵向线、删除单条线、重置切图线和切片数量预览。
- Codex App 插件文档：新增插件安装教程，区分 AI 自动安装、手动安装和本仓库开发调试安装；同步优化插件 README 与技能说明。
- 本地 Codex 连接画布文档：新增 Canvas Agent、MCP、SSE 与浏览器画布交互原理说明，需确认文档导航和描述是否清晰。
- 顶部导航：缩小桌面顶部导航栏高度，导航项下划线和内容垂直对齐需确认。
- 配置与用户偏好：新增独立顶部导航页面，并在页面和弹窗中复用同一套配置面板；新增 Codex Tab，展示插件安装和本地 Agent 启动步骤，可配置本地 Agent 地址、Connect token 和工具执行确认开关。
- Codex 全局连接：配置页可直接连接本地 Canvas Agent，连接状态同步到画布和顶部 OpenAI 状态图标，刷新后会按本地保存的地址和 token 自动尝试重连；需验证已连接、连接中、连接失败、断开和刷新自动重连状态。
- 画布 Agent：Codex 旧 thread 在重启后不可用时会自动新建会话并重试，需验证进入画布直接发送消息不会再出现 `thread not loaded` 或 `no rollout found`。
- 画布 Agent：切换不同画布时会清空并重新加载当前 canvasId 对应的 Codex 会话，需验证不会继续显示上一个画布的对话。
- 画布 Agent：右侧 Agent 面板移除网站 / 本机模式切换，固定使用本机 Agent。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- Gemini 生图：生图工作台和画布生图会把尺寸配置转换为 Gemini `responseFormat.image.aspectRatio` 传入，需验证 1:1、9:16、16:9 及自定义像素尺寸的输出比例。

- 提示词库：移除 Next.js route 抓取和服务端内存缓存，改为浏览器前端直连 GitHub 原始文件并缓存到 IndexedDB。

- 前端构建：移除 Next.js，改为 Vite + React Router，并将前端目录调整为 `pages` / `components` / `stores` / `lib` / `layouts` / `styles`，需验证首页、图片/视频工作台、素材、提示词库和画布路由跳转。
