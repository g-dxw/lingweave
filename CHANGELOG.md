# CHANGELOG

## Unreleased

+ [修复] 优化移动端顶部导航、首页案例卡片、Agent 面板、画布工具栏及生成与配置页面的窄屏布局，避免控件叠压、逐字换行和整页横向溢出。
+ [修复] GitHub Pages 在代码合并到 `main` 后自动发布，避免 PR 合并后线上版本仍停留在旧构建。
+ [新增] 画布增加 AI 思维节点，支持自由发散、连贯序列和总分展开，并生成可继续连接现有生成配置的结构化文本子节点。
+ [修复] AI 思维节点兼容从 Responses 流事件返回的函数调用结果，避免最终响应缺少 output 时误报未返回结构化结果。
+ [修复] 画布 Agent 对同一运行中节点的重复生成请求会直接复用当前状态，避免连续工具调用创建重叠结果节点。
+ [调整] 生成配置按任务说明、参考素材和引用文本确定性组装最终提示词，并避免图片请求重复追加引用编号。
+ [新增] 画布空白区域支持双击打开节点选择菜单，并在点击位置创建节点。
+ [优化] 更新 LingWeave 品牌图标，以交织双线表现连续创作与无限画布，并统一主站、文档和 Codex 插件图标。
+ [优化] 提示词库图片改由 GitHub Release 资源包分发，避免在代码仓库中提交大量生成封面，并保留上游同步兜底。
+ [修复] 改用 nanoid 生成节点与资源 ID，避免部分环境缺少 `crypto.randomUUID` 时页面白屏。

## v0.1.0

+ [新增] LingWeave（灵织）以独立项目重新建立版本基线，并保留原项目提交历史、来源说明与 AGPL-3.0 协议。
+ [新增] 提供节点式无限画布，支持文本、图片、视频、音频、生成配置和分组节点，以及连线、缩放、撤销重做和导入导出。
+ [新增] 提供生图与视频工作台、提示词库、素材管理和连续内容生成流程，支持从已有结果继续引用和分支创作。
+ [新增] 支持可选 Niffler 登录与用户自定义 OpenAI / Gemini 兼容渠道，本地开发默认通过 Vite 代理连接 Niffler。
+ [新增] 提供本地 Canvas Agent 与 Codex App 插件，可读取画布、创建节点、连接流程、切换页面并触发生成。
+ [新增] 前端接入 Vitest 与 GitHub Actions，覆盖输入法、配置、默认案例、提示词库和画布生成核心逻辑。
+ [调整] 画布、素材和生成记录默认保存在浏览器本地，支持导入导出及可选 WebDAV 同步。
+ [优化] 页面与 Agent 按需加载，画布内容按项目独立防抖持久化，并清理未使用依赖。
+ [修复] 修复连续生成误带下游提示词、输入框焦点反馈、默认模型保存和 GitHub Pages 子路径资源等问题。
